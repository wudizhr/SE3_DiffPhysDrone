# 配置说明

本项目已经把训练和可视化参数统一到 YAML 配置。核心目标是让训练时使用的环境、模型和推理参数能随 checkpoint 一起保存，并被可视化自动复用。

## 配置优先级

参数合并顺序如下，后面的值会覆盖前面的值：

1. dataclass 默认值
2. `configs/default.yaml`
3. 用户传入的 YAML
4. checkpoint 目录中的 `config.yaml`
5. 当前可视化 YAML 中显式写出的字段
6. 当前入口脚本的固定参数，例如 `--config` 指向的配置文件路径

训练时推荐：

```bash
python3 train/main_cuda.py --config configs/single_agent.yaml
```

可视化时推荐：

```bash
python3 visualization/export_scene_rollout.py --config visualization/config/scene_rollout_example.yaml
```

## `env`

环境和随机场景生成参数。训练、场景导出、rollout 导出都会使用这些字段。

常用字段：

- `width` / `height`：深度图分辨率。
- `grad_decay`：环境状态梯度衰减。
- `fov_x_half_tan`：相机水平半视场角的 tan 值。
- `single`：是否使用单机模式。
- `gate`、`ground_voxels`、`scaffold`：障碍物类型开关。
- `speed_mtp`：速度倍率，会影响目标速度和场景尺度。
- `random_rotation`：是否随机初始姿态。
- `cam_angle`：相机俯仰安装角，单位为度。
- `gap` / `gap_prob`：窄缝墙场景开关和概率。
- `n_balls`、`n_voxels`、`n_cyl`、`n_cyl_h`、`n_ground_voxels`：各类障碍物数量。
- `quad_mass`：无人机标称质量，单位 kg。当前 point-mass 后端不使用该字段，后续 CTBR 后端会用 `env.mass` 计算推力加速度。
- `quad_mass_randomization`：训练时质量随机化比例，例如 `0.05` 表示每次 reset 在 `quad_mass * [0.95, 1.05]` 内采样；设为 `0.0` 时固定质量。
- `ctbr_body_rate_limit`：CTBR action adapter 的机体系角速度限幅，单位 rad/s。
- `ctbr_thrust_min` / `ctbr_thrust_max`：CTBR collective thrust 命令范围，单位 N。
- `ctbr_omega_time_constant` / `ctbr_thrust_time_constant`：CTBR 一阶响应时间常数，单位 s。
- `ctbr_linear_drag`：CTBR PyTorch 参考动力学中的线性阻力系数。

## `train`

训练循环参数。

- `resume`：恢复训练的 checkpoint 路径。
- `save_dir`：输出根目录，默认当前项目目录。
- `num_envs`：并行环境数量，也是 CUDA batch 维度；训练时会同时 rollout 这么多份环境。
- `batch_size`：旧字段兼容入口，语义等同于 `num_envs`；新配置建议只写 `num_envs`。
- `num_iters`：训练迭代次数。
- `lr`：学习率。
- `timesteps`：每次 rollout 的控制步数。
- `yaw_drift`：训练时是否加入 yaw 漂移扰动。
- `use_future_collision_samples`：避障/碰撞 loss 是否沿当前速度外推 10 个短时刻查询最近距离；默认为 `true`，保持原有训练逻辑。设为 `false` 时只用当前真实位置查询最近距离。

## `loss`

训练损失权重。

- `coef_v`：速度跟踪损失权重。
- `coef_v_pred`：速度预测辅助损失权重。
- `coef_collide`：碰撞损失权重。
- `coef_obj_avoidance`：障碍物 clearance 损失权重。
- `coef_d_acc`：控制加速度正则权重。
- `coef_d_jerk`：控制 jerk 正则权重。

## `inference`

训练和离线 rollout 都需要对齐的推理参数。

- `device`：推理设备，通常是 `cuda`。
- `no_odom`：是否不使用里程计速度输入。
- `yaw_target_correction`：是否用目标方向修正 yaw。
- `ctl_freq`：控制频率，单位 Hz。
- `max_steps`：离线 rollout 最大步数。
- `frame_id`：导出和播放使用的坐标系名。
- `deterministic_visualization`：可视化 policy rollout 时是否关闭运行时随机化。默认 `true`，训练不受影响。
- `record_mid360`：导出 policy rollout 时是否同时渲染并保存 MID360 点云。开启后 `.npz` 会包含 `mid360_points` 和 `mid360_ranges`。
- `record_mid360_pseudo_image`：导出 policy rollout 时是否保存 MID360 策略实际输入 CNN 的距离伪图像。MID360 策略默认开启，RViz 播放时发布 `/diffphys/mid360_pseudo_image` 和 `/diffphys/mid360_pseudo_image_viz`。
- `mid360_points_per_scan`：每帧点云射线数量，示例使用 `20000`。
- `mid360_vertical_channels`：垂直采样线数量，示例使用 `64`。
- `mid360_min_range` / `mid360_max_range`：雷达盲区和最大量程，示例为 `0.1` / `70.0` 米。
- `mid360_vertical_min_deg` / `mid360_vertical_max_deg`：垂直视场，示例为 `-7.0` 到 `52.0` 度。

## `model`

模型元信息。训练脚本会在 checkpoint 的 `config.yaml` 和 `model_info.yaml` 中写入实际值。

- `name`：模型工厂名称。当前可用值为 `pm_model`。`se3_model` 暂未接入 CTBR 控制策略。
- `model_type`：模型类型标识。
- `model_class`：模型类名。
- `backend_name`：动力学后端名称。当前可用 `point_mass` 和 `ctbr`；`ctbr` 是第一版 PyTorch 参考实现。
- `dim_obs`：状态输入维度。
- `dim_action`：动作输出维度。
- `depth_pool_kernel`：深度图池化核大小。
- `hidden_dim`：模型隐藏维度，供后续模型使用。
- `action_mode`：动作语义。当前 `pm_model` 和 `mid360_cnn_model` 使用 `accel_velocity`，由 `control/ActionAdapter` 统一转换成环境控制量。`ctbr` 会把 4 维 raw action 转成 `[collective_thrust_N, wx, wy, wz]`。

通常不需要手动改 `model`，除非你切换模型结构。

## `sensor`

传感器输入和 observation 构造配置。

- `name`：传感器/observation 构造器名称。当前可用 `depth_odom`，`mid360` 已提供点云伪图像预处理方法。
- `depth_pool_kernel`：深度图池化核大小。
- `use_depth`：是否使用深度图。
- `use_odom`：是否把当前速度加入状态输入。
- `use_flow`：是否使用光流，当前未接入训练主流程。
- `use_mid360`：是否使用 MID360 点云，当前作为配置标记保留。
- `mid360_theta_min_deg` / `mid360_theta_max_deg`：伪图像方位角覆盖范围。
- `mid360_phi_min_deg` / `mid360_phi_max_deg`：伪图像极角覆盖范围，`phi` 是相对 Z 轴的极角。
- `mid360_theta_resolution_deg` / `mid360_phi_resolution_deg`：伪图像角分辨率。
- `mid360_encoding`：MID360 observation 表达方式，当前实现 `pseudo_image`。

后续接入 MID360 时，应在 `sensors/` 下实现点云预处理和 observation 构造，不要把点云逻辑直接写进训练脚本。

## `paths`

可视化和导出路径。

- `scene_path`：`.pt` 场景快照路径。
- `rollout_path`：`.npz` rollout 输出或播放路径。
- `checkpoint_path`：策略 checkpoint 路径，可选。未指定时，policy rollout 会自动查找项目 `checkpoints/` 目录下修改时间最新的 `checkpoint_final.pth`，并读取该目录中的训练参数。

相对路径会按配置文件所在目录解析。

## `scene`

场景导出专用配置。

常用字段：

- `seed`：场景随机种子。
- `device`：生成场景使用的设备。
- `mission.start`：起点。
- `mission.target`：终点。
- `mission.target_reached_radius`：认为到达目标的半径。
- `clearance.enabled`：是否清理起点/终点附近障碍物。
- `clearance.radius`：清理半径。
- `obstacles.*`：障碍物随机采样范围。

## `playback`

RViz2 离线播放参数。

- `playback_speed`：播放倍速。
- `publish_prefix`：ROS topic 前缀。
- `body_axis_length` / `body_axis_radius`：机体系坐标轴 marker 尺寸。
- `loop`：是否循环播放。
- `exit_on_finish`：播放结束后是否退出。
- `frame_id`：播放坐标系。
- `rate_hz`、`ctl_freq`、`ctl_dt`：播放频率设置，优先级为 `ctl_freq` > `rate_hz` > `ctl_dt` > rollout 内元数据。

## 推荐修改方式

训练实验只改：

```text
configs/single_agent.yaml
```

可视化实验只改：

```text
visualization/config/scene_rollout_example.yaml
```

不要同时在多个文件里改同一个参数。训练入口不再支持 `--num_iters`、`--ctl_freq`、`--resume` 等散落命令行参数；需要临时实验时，新建或复制一个 YAML 配置文件：

```bash
cp configs/single_agent.yaml configs/debug.yaml
python3 train/main_cuda.py --config configs/debug.yaml
```
